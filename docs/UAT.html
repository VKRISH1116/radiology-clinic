UAT & Test Traceability

Does the built system actually satisfy every acceptance criterion we signed off in the PRD? This page traces each AC to the automated test(s) that prove it, records the manual User-Acceptance walkthrough, and — honestly — flags the criteria the MVP build does not yet meet.

Version: 0.1 Date: 2026-07-16 Author: Yatin Krishna Traces from: PRD v0.2 (AC-F1-1 … AC-F8-2) Phase: 7 · Testing
What UAT is, and why this page exists. Unit and integration tests answer "did I build the thing right?" — small, developer-facing checks. User-Acceptance Testing answers a different question: "did I build the right thing?" — measured against the acceptance criteria the business signed off in the PRD, not against the code. The PRD promised that its Given/When/Then criteria "become the test cases in Phase 7" — so this page closes that loop. A traceability matrix maps every promise to its proof (or its absence), so nothing is silently dropped and "done" is demonstrated, not asserted.

Why the pyramid shape (many backend, fewer E2E). Backend tests are fast and precise — they pin down exact status codes and money maths, so we write lots. E2E tests are slow and brittle (they spin up a browser and a whole stack) but are the only ones that prove the pieces actually talk to each other — so we keep a thin layer of the highest-value journeys. This is the standard "test pyramid".

3Manual UAT script

The walkthrough a human runs (or shows the mentor) to sign off the release by hand — the automated tests prove the rules, this proves the experience. Pre-req: backend on :8080, Postgres up, SPA on :5173. Seeded admin: admin@clinic.local.

  1. Register & land. Open the SPA → Register with a fresh email + an 8+ char password → expect to be logged in and dropped on the patient dashboard. AC-F1-1
  2. Book two studies. Book a scan → tick two studies → confirm the running bill equals their sum → pick a free slot → Confirm → the new appointment shows as BOOKED. AC-F2-2
  3. Capacity. On a busy date, confirm fully-booked slots render greyed/disabled and can't be selected. AC-F2-1
  4. Reschedule & cancel. Reschedule the appointment to another slot (old time is released) → then cancel it → row shows CANCELLED. AC-F2-3
  5. Role gate. While logged in as the patient, try to reach /staff or /admin → you are redirected away. AC-F1-2
  6. Staff completes & uploads. Log in as staff → open today's schedule → Mark complete a booked visit → upload a PDF report (try a non-PDF first: rejected). AC-F4-1 / AC-F5-2
  7. Report privacy. Back as the owning patient, download the report. As a different patient, confirm you cannot. AC-F4-2
  8. Referral maths. As admin, open Referrals → confirm a completed referred visit shows the expected payout (default 20%, or the doctor's specific rate if a rule applies) → Mark paid → status flips to PAID. AC-F6-1/2/3
  9. Catalogue & snapshot. As admin, change a study's price → book a new appointment (uses new price) → confirm an earlier appointment still shows its old billed amount. Deactivate a study → confirm it disappears from the booking list. AC-F8-1 / AC-F8-2

4Gaps & decisions

UAT's real value is surfacing the honest delta between "signed off" and "built". The consent-gate gap it first flagged (AC-F3-1) has since been fixed — a required consent checkbox now blocks registration until it's ticked. Three criteria remain not fully met; none blocks the MVP demo, and each has a decision attached.

ItemStatusDecision / next step
AC-F1-3 Password resetDeferred Already deferred at URD/PRD sign-off (needs an email transport we're not wiring for MVP). Backlog for a post-MVP iteration.
AC-F5-1 Editable billPartial Bill is auto-computed from snapshotted study prices. Staff-editable billing (discounts/adjustments) not in MVP — confirm with mentor whether it's needed for v1.
AC-F7-1 Admin KPI dashboardGap The data exists across tabs but there's no summary-counts header. Needs one small aggregate endpoint + a KPI tile row. Candidate for Phase 7 polish or an early Phase 8 item.

5How to run the tests

Backend (73 tests, H2 in-memory — no Postgres needed):

& "C:\tools\apache-maven-3.9.16\bin\mvn.cmd" -f backend\pom.xml test

Frontend component (18 tests):

cd frontend && npm test

End-to-end (needs backend :8080 + Postgres running; Vite auto-starts):

cd frontend && npm run test:e2e
